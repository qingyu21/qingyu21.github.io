CMake Examples笔记（一）基础

0. 前言

本文是CMake Exampe的学习笔记。

具体的例子可以等用到的时候再仔细看。

1. Hello CMake

  • CMakeLists.txt: Contains the CMake commands you wish to run
  • main.cpp: A simple “Hello World” cpp file.
# Set the minimum version of CMake that can be used
# To find the cmake version run
# $ cmake --version
cmake_minimum_required(VERSION 3.5)

# Set the project name
project (hello_cmake)

# Add an executable
add_executable(hello_cmake main.cpp)

一个常见的做法是将project name和executable name设置为同一个：

cmake_minimum_required(VERSION 2.6)
# the project() function create a variable ${PROJECT_NAME} with the value hello_cmake.
project (hello_cmake)
add_executable(${PROJECT_NAME} main.cpp)

Binary Directory

CMAKE_BINARY_DIR：运行cmake命令的根目录，它同时也是所有生成的二进制文件的根目录。

  • In-Place Build：在与源代码相同的目录结构中生成所有临时构建文件。（源文件和构建文件穿插在一起，所以比较乱）
  • Out-of-Source Build：新建一个build文件夹，并在其中运行cmake命令，参数为源文件所在的目录。（源代码树比较清楚，是推荐的方式）

Building the Examples

mkdir build
cd build
cmake ..

2. Hello Headers

将源文件和头文件放到不同的文件夹（src、include）中保存：

  • CmakeLists.txt: Contains the CMake commands you wish to run.
  • include/Hello.h: The header file to include.
  • src/Hello.cpp: A source file to compile.
  • src/main.cpp: The source file with main.
# Set the minimum version of CMake that can be used
# To find the cmake version run
# $ cmake --version
cmake_minimum_required(VERSION 3.5)

# Set the project name
project (hello_headers)

# Create a sources variable with a link to all cpp files to compile
set(SOURCES
    src/Hello.cpp
    src/main.cpp
)

# Add an executable with the above sources
add_executable(hello_headers ${SOURCES})

# Set the directories that should be included in the build command for this target
# when running g++ these will be included as -I/directory/path/
target_include_directories(hello_headers
    PRIVATE 
        ${PROJECT_SOURCE_DIR}/include
)

概念

Directory Paths

CMake限定了一些变量，可用于寻找项目中有用的目录。

Variable Info
CMAKE_SOURCE_DIR The root source directory
CMAKE_CURRENT_SOURCE_DIR The current source directory if using sub-projects and directories.
PROJECT_SOURCE_DIR The source directory of the current cmake project.
CMAKE_BINARY_DIR The root binary / build directory. This is the directory where you ran the cmake command.
CMAKE_CURRENT_BINARY_DIR The build directory you are currently in.
PROJECT_BINARY_DIR The build directory for the current project.

Source Files Variable

可通过set()来将一些源文件包含到一个变量中。

An alternative to setting specific file names in the SOURCES variable is to use a GLOB command to find files using wildcard pattern matching.

file(GLOB SOURCES "src/*.cpp")

For modern CMake it is NOT recommended to use a variable for sources. Instead it is typical to directly declare the sources in the add_xxx function.

This is particularly important for glob commands which may not always show you the correct results if you add a new source file.

Including Directories

target_include_directories()可以使编译器意识到不同的include文件夹。

编译时，这些文件夹被添加到编译命令上：-I/directory/path。

The PRIVATE identifier specifies the scope of the include. This is important for libraries and is explained in the next example. More details on the function is available here

Verbose Output

In the previous examples, when running the make command the output only shows the status of the build. To see the full output for debugging purposes you can add VERBOSE=1 flag when running make.

The VERBOSE output is show below, and a examination of the output shows the include directories being added to the c++ compiler command.

3. Static Library

创建并链接一个静态库。本例子仅仅将库文件和二进制文件放在同一个文件夹中，通常它们是放在子项目中的。

  • CMakeLists.txt: Contains the CMake commands you wish to run
  • include/static/Hello.h: The header file to include
  • src/Hello.cpp: A source file to compile
  • src/main.cpp: The source file with main
cmake_minimum_required(VERSION 3.5)

project(hello_library)

############################################################
# Create a library
############################################################

#Generate the static library from the library sources
#library name: libhello_library.a
add_library(hello_library STATIC 
    src/Hello.cpp
)

target_include_directories(hello_library
    PUBLIC 
        ${PROJECT_SOURCE_DIR}/include
)


############################################################
# Create an executable
############################################################

# Add an executable with the above sources
add_executable(hello_binary 
    src/main.cpp
)

# link the new hello_library target with the hello_binary target
target_link_libraries( hello_binary
    PRIVATE 
        hello_library
)

概念

Adding a Static Library

add_library()用于通过一些源文件来创建一个库文件。

As mentioned in the previous example, we pass the source files directly to the add_library call, as recommended for modern CMake.

Populating Including Directories

我们在target_include_directories中将范围设置为PUBLIC。

This will cause the included directory used in the following places:

  • When compiling the library

  • When compiling any additional target that links the library.

The meaning of scopes are:

  • PRIVATE - the directory is added to this target’s include directories
  • INTERFACE - the directory is added to the include directories for any targets that link this library.
  • PUBLIC - As above, it is included in this library and also any targets that link this library.

The meaning of scopes are:

  • PRIVATE - the directory is added to this target’s include directories
  • INTERFACE - the directory is added to the include directories for any targets that link this library.
  • PUBLIC - As above, it is included in this library and also any targets that link this library.

Linking a Library

使用target_link_libraries()来告诉编译器将此库文件链接到可执行文件。

链接的过程中，也会将带有PUBLIC和INTERFACE的include文件夹的作用域传播下去。

4. Shared Library

创建并链接一个动态库。

本例子同样展示如何创建一个alias target。

  • CMakeLists.txt: Contains the CMake commands you wish to run
  • include/shared/Hello.h: The header file to include
  • src/Hello.cpp: A source file to compile
  • src/main.cpp: The source file with main
cmake_minimum_required(VERSION 3.5)

project(hello_library)

############################################################
# Create a library
############################################################

#Generate the shared library from the library sources
#library name: libhello_library.so
add_library(hello_library SHARED 
    src/Hello.cpp
)
add_library(hello::library ALIAS hello_library)

target_include_directories(hello_library
    PUBLIC 
        ${PROJECT_SOURCE_DIR}/include
)

############################################################
# Create an executable
############################################################

# Add an executable with the above sources
add_executable(hello_binary
    src/main.cpp
)

# link the new hello_library target with the hello_binary target
target_link_libraries( hello_binary
    PRIVATE 
        hello::library
)

概念

Adding a Shared Library

利用add_library()来创建一个动态库。

Alias Target

As the name suggests an alias target is an alternative name for a target that can be used instead of the real target name in read-only contexts.

Linking a Shared Library

target_link_library()

5. Installing

创建一个make install目标，来将文件和二进制安装到系统中。

  • CMakeLists.txt: Contains the CMake commands you wish to run
  • cmake-examples.conf: An example configuration file
  • include/installing/Hello.h: The header file to include
  • src/Hello.cpp: A source file to compile
  • src/main.cpp: The source file with main
cmake_minimum_required(VERSION 3.5)

project(cmake_examples_install)

############################################################
# Create a library
############################################################

#Generate the shared library from the library sources
add_library(cmake_examples_inst SHARED
    src/Hello.cpp
)

target_include_directories(cmake_examples_inst
    PUBLIC 
        ${PROJECT_SOURCE_DIR}/include
)

############################################################
# Create an executable
############################################################

# Add an executable with the above sources
add_executable(cmake_examples_inst_bin
    src/main.cpp
)

# link the new hello_library target with the hello_binary target
target_link_libraries( cmake_examples_inst_bin
    PRIVATE 
        cmake_examples_inst
)

############################################################
# Install
############################################################

# Binaries
install (TARGETS cmake_examples_inst_bin
    DESTINATION bin)

# Library
# Note: may not work on windows
install (TARGETS cmake_examples_inst
    LIBRARY DESTINATION lib)

# Header files
install(DIRECTORY ${PROJECT_SOURCE_DIR}/include/ 
    DESTINATION include)

# Config
install (FILES cmake-examples.conf
    DESTINATION etc)

概念

Installing

CMake提供了添加make instlal目标的能力，允许用户安装二进制文件、库和其他文件。

cmake .. -D CMAKE_INSTALL_PREFIX=/install/location

通过install()来安装文件。

After make install has been run, CMake generates an install_manifest.txt file which includes details on all installed files.

If you run the make install command as root, the install_manifest.txt file will be owned by root.

6. Build Type

CMake有一些内置的构建配置，可以用来编译你的项目。这些配置指定了优化级别以及调试信息是否包含在二进制文件中。

提供的构建级别：

  • Release - Adds the -O3 -DNDEBUG flags to the compiler
  • Debug - Adds the -g flag
  • MinSizeRel - Adds -Os -DNDEBUG
  • RelWithDebInfo - Adds -O2 -g -DNDEBUG flags

概念

Set Build Type

cmake .. -DCMAKE_BUILD_TYPE=Release

Set Default Build Type

CMake提供的默认构建类型是不包含用于优化的编译器标志。

可以设置一个默认的构建类型：在顶层CMakeLists.txt中添加：

if(NOT CMAKE_BUILD_TYPE AND NOT CMAKE_CONFIGURATION_TYPES)
  message("Setting build type to 'RelWithDebInfo' as none was specified.")
  set(CMAKE_BUILD_TYPE RelWithDebInfo CACHE STRING "Choose the type of build." FORCE)
  # Set the possible values of build type for cmake-gui
  set_property(CACHE CMAKE_BUILD_TYPE PROPERTY STRINGS "Debug" "Release"
    "MinSizeRel" "RelWithDebInfo")
endif()

7. Compile Flags

CMake支持以多种方式来设置compile flags：

  • target_compile_definitions()
  • 使用变量CMAKE_C_FLAGS和CMAKE_CXX_FLAGS

文件：

  • CmakeLists.txt: Contains the CMake commands you wish to run
  • main.cpp: The source file with main
cmake_minimum_required(VERSION 3.5)

# Set a default C++ compile flag
set (CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -DEX2" CACHE STRING "Set C++ Compiler Flags" FORCE)

# Set the project name
project (compile_flags)

# Add an executable
add_executable(cmake_examples_compile_flags main.cpp)

target_compile_definitions(cmake_examples_compile_flags 
    PRIVATE EX3
)

概念：

Set Per-Target C++ Flags

Modern CMake中推荐的设置C++ flags的方法是使用per-target flags，可以通过target_compile_definitions将它们填入到其它的targets，这将为库填充INTERFACE_COMPILE_DEFINITIONS，并根据scope将definition推送到链接的target。

# This will cause the compiler to add the definition -DEX3 when compiling the target.
target_compile_definitions(cmake_examples_compile_flags
    PRIVATE EX3
)

If the target was a library and the scope PUBLIC or INTERFACE has been chosen the definition would also be included in any executables that link this target.

For compiler options you can also use the target_compile_options() function.

Set Default C++ Flags

默认的CMAKE_CXX_FLAGS要么是空的，要么包含适合Build Type的flags。

# set additional default compile flags
# add to your top level CMakeLists.txt
set (CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -DEX2" CACHE STRING "Set C++ Compiler Flags" FORCE)

类似的变量：

  • Setting C compiler flags using CMAKE_C_FLAGS
  • Setting linker flags using CMAKE_LINKER_FLAGS.

The values CACHE STRING "Set C++ Compiler Flags" FORCE from the above command are used to force this variable to be set in the CMakeCache.txt file.

For more details, see here

一旦设置了CMAKE_C_FLAGS和CMAKE_CXX_FLAGS，就会为这个目录或任何包含的子目录中的所有目标全局设置一个编译器标志/定义。现在不推荐一般使用这种方法，最好使用target_compile_definitions()函数。

Set CMake Flags

类似于build type，我们可以通过以下方式来设置一个global C++ compiler flag。

gui工具或者cmake .. -DCMAKE_CXX_FLAGS="-DEX3"

8. Third Party Library

几乎大多数重要项目都会有包含第三方库、头文件、程序的需求。

CMake提供了find_package()函数来找到这些工具的路径，它将以FindXXX.cmake的格式从CMAKE_MODULE_PATH中的文件夹列表中搜索CMake模块。

在Linux中，默认的search path包含/usr/share/cmake/Modules。

  • CmakeLists.txt: Contains the CMake commands you wish to run
  • main.cpp: The source file with main

本例子需要boost库的安装。

cmake_minimum_required(VERSION 3.5)

# Set the project name
project (third_party_include)


# find a boost install with the libraries filesystem and system
find_package(Boost 1.46.1 REQUIRED COMPONENTS filesystem system)

# check if boost was found
if(Boost_FOUND)
    message ("boost found")
else()
    message (FATAL_ERROR "Cannot find Boost")
endif()

# Add an executable
add_executable(third_party_include main.cpp)

# link against the boost libraries
target_link_libraries( third_party_include
    PRIVATE
        Boost::filesystem
)

概念

Finding a Package

The exact format of the arguments to find_package will depend on the module you are looking for. This is typically documented at the top of the FindXXX.cmake file.

Checking if the package is found

Most included packages will set a variable XXX_FOUND, which can be used to check if the package is available on the system.

Exported Variables

After a package is found it will often export variables which can inform the user where to find the library, header, or executable files.

Alias / Imported targets

Most modern CMake libraries export ALIAS targets in their module files.

The benefit of imported targets are that they can also populate include directories and linked libraries.

Non-alias targets

  • xxx_INCLUDE_DIRS A variable pointing to the include directory for the library.
  • xxx_LIBRARY A variable pointing to the library path.
# Include the boost headers
target_include_directories( third_party_include
    PRIVATE ${Boost_INCLUDE_DIRS}
)

# link against the boost libraries
target_link_libraries( third_party_include
    PRIVATE
    ${Boost_SYSTEM_LIBRARY}
    ${Boost_FILESYSTEM_LIBRARY}
)

9. Compiling with clang

  • CmakeLists.txt: Contains the CMake commands you wish to run
  • main.cpp: A simple “Hello World” cpp file.

概念

Compiler Option

CMake暴露了一些选项，以控制用于编译和链接你的代码的程序。

  • CMAKE_C_COMPILER The program used to compile c code.
  • CMAKE_CXX_COMPILER The program used to compile c++ code.
  • CMAKE_LINKER The program used to link your binary.

Setting Flags

you can set CMake options using either a cmake gui or by passing from the command line.

cmake .. -DCMAKE_C_COMPILER=clang-3.6 -DCMAKE_CXX_COMPILER=clang++-3.6

10. Building with ninja

Cmake是元构建系统，本例尝试使用ninja作为构建工具。

  • CmakeLists.txt: Contains the CMake commands you wish to run
  • main.cpp: A simple “Hello World” cpp file.

概念

Generators

CMake generators 负责向下层的构建系统的输入文件（比如Makefiles）中写入内容。通过cmake --help可以查看可用的generator。

Command-Line Build Tool Generators

用于command-line build tools，如Make和Ninja。

构建工具链必须在利用Cmake生成构建系统之前配置。

IDE Build Tool Generators

用于那些包含自己的编译器的IDE，如Visual Studio和Xcode。

Extra Generators

These are generators create a configuration to work with an alternative IDE tool and must be included with either an IDE or Command-Line generator.

Calling a Generator

# To call a CMake generator you can use the -G command line switch, for example:
cmake .. -G Ninja

# After doing the above CMake will generate the required Ninja build files, which can be run from using the ninja command.
cmake .. -G Ninja

11. Import Targets

新版本的Cmake允许通过使用imported ALIAS targets来链接第三方库。

  • CmakeLists.txt: Contains the CMake commands you wish to run
  • main.cpp: The source file with main

Imported targets are read-only targets that are exported by FindXXX modules.

# automtaically link the Boost::filesystem and Boost::system libraries while also including the Boost include directories.
target_link_libraries( imported_targets
      PRIVATE
          Boost::filesystem
  )

12. CPP Standard

展示了设置C++标准的不同方式，以及不同的Cmake版本。

C++ Standard Common Method

可以通过大多数版本的Cmake来使用。

  • CmakeLists.txt: Contains the CMake commands you wish to run
  • main.cpp: A simple “Hello World” cpp file targeting C++11
# Set the minimum version of CMake that can be used
# To find the cmake version run
# $ cmake --version
cmake_minimum_required(VERSION 2.8)

# Set the project name
project (hello_cpp11)

# try conditional compilation
include(CheckCXXCompilerFlag)
CHECK_CXX_COMPILER_FLAG("-std=c++11" COMPILER_SUPPORTS_CXX11)
CHECK_CXX_COMPILER_FLAG("-std=c++0x" COMPILER_SUPPORTS_CXX0X)

# check results and add flag
if(COMPILER_SUPPORTS_CXX11)#
    set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -std=c++11")
elseif(COMPILER_SUPPORTS_CXX0X)#
    set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -std=c++0x")
else()
    message(STATUS "The compiler ${CMAKE_CXX_COMPILER} has no C++11 support. Please use a different C++ compiler.")
endif()

# Add an executable
add_executable(hello_cpp11 main.cpp)

概念

Checking Compile flags

CMake支持尝试用你传递给函数CHECK_CXX_COMPILER_FLAG的任何标志来编译程序。然后，结果被存储在你传入的变量中。

include(CheckCXXCompilerFlag)告诉CMake包含这个函数，使其可以使用。

Adding the flag

Once you have determined if the compile supports a flag, you can then use the standard cmake methods to add this flag to a target.

Set CXX Standard

使用CMAKE_CXX_STANDARD变量来设置C++标准，需要的最小Cmake版本为v3.1。

  • CmakeLists.txt: Contains the CMake commands you wish to run
  • main.cpp: A simple “Hello World” cpp file targeting C++11
# Set the minimum version of CMake that can be used
# To find the cmake version run
# $ cmake --version
cmake_minimum_required(VERSION 3.1)

# Set the project name
project (hello_cpp11)

# set the C++ standard to C++ 11
set(CMAKE_CXX_STANDARD 11)

# Add an executable
add_executable(hello_cpp11 main.cpp)

概念

Using CXX_STANDARD property

设置CMAKE_CXX_STANDARD变量会导致所有targets的CXX_STANDARD属性被设置。这导致在编译时Cmake会设置合适的flag。

The CMAKE_CXX_STANDARD variable falls back to the closest appropriate standard without a failure. For example, if you request -std=gnu11 you may end up with -std=gnu0x.

This can result in an unexpected failure at compile time.

Set C++ Standard

使用target_compile_features函数来设置C++标准，需要的最小Cmake版本为v3.1。

  • CmakeLists.txt: Contains the CMake commands you wish to run
  • main.cpp: A simple “Hello World” cpp file targeting C++11
# Set the minimum version of CMake that can be used
# To find the cmake version run
# $ cmake --version
cmake_minimum_required(VERSION 3.1)

# Set the project name
project (hello_cpp11)

# Add an executable
add_executable(hello_cpp11 main.cpp)

# set the C++ standard to the appropriate standard for using auto
target_compile_features(hello_cpp11 PUBLIC cxx_auto_type)

# Print the list of known compile features for this version of CMake
message("List of compile features: ${CMAKE_CXX_COMPILE_FEATURES}")

概念

Using target_compile_features

Calling the target_compile_features function on a target will look at the passed in feature and determine the correct compiler flag to use for your target.
