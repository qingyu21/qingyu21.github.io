C++多线程编程

0. 时间

在<chrono>中，标准库提供了处理时间的设施。

0.1 特性

明确区分时间点与时间段，明确区分不同的时间单位。

  • 事件点类型：chrono::steady_clock::time_point
  • 时间段类型：chrono::milliseconds，chrono::seconds，chrono::minutes

方便的运算符重载：时间点+时间段=时间点，时间点-时间点=时间段

0.2 例子

1. 时间点计算差值得到时间段

auto t0 = std::chrono::steady_clock::now(); // time_point类型
for (volatile int i = 0; i < 10000000; i++); 
auto t1 = std::chrono::steady_clock::now();
auto dt = t1 - t0; // duration类型
int64_t ms = std::chrono::duration_cast<std::chrono::milliseconds>(dt).count(); // 统一单位

2. 用double类型表示的时间段：

duration<T, R> 表示用 T 类型表示，且时间单位是 R，R 省略不写就是秒，std::milli 就是毫秒，std::micro 就是微秒

using double_ms = std::chrono::duration<double, std::milli>;
double ms = std::chrono::duration_cast<double_ms>(dt).count();

（seconds 是 duration<int64_t> 的类型别名，milliseconds 是 duration<int64_t, std::milli> 的类型别名）

3. 线程休眠

std::this_thread::sleep_for(std::chrono::milliseconds(400)); // 睡眠一个时间段
// 睡眠到一个时间点
auto t = std::chrono::steady_clock::now() + std::chrono::milliseconds(400);
std::this_thread::sleep_until(t);
std::this_thread::sleep(10ms+33us) // 时间后缀 std::chrono_literals

1. 线程

1.1 创建并等待线程结束

void download(std::string file) {
    for (int i = 0; i < 10; i++) {
        std::cout << "Downloading " << file
                  << " (" << i * 10 << "%)..." << std::endl;
        std::this_thread::sleep_for(std::chrono::milliseconds(400));
    }
    std::cout << "Download complete: " << file << std::endl;
}

void interact() {
    std::string name;
    std::cin >> name;
    std::cout << "Hi, " << name << std::endl;
}

int main() {
    // 启动一个线程
    std::thread t1([&] {
        download("hello.zip");
    });
    interact();
    std::cout << "Waiting for child thread..." << std::endl;
    t1.join(); // 如果没有这一行，那么还没等子线程结束就退出了 
    std::cout << "Child thread exited!" << std::endl;
    return 0;
}

1.2 管理线程生命周期

1. 线程RAII

作为C++类，thread自定义了解构函数，删除了拷贝构造/函数，但是提供了移动构造/赋值函数。

void myfunc() {
    std::thread t1([&] {
        download("hello.zip");
    });
    // 退出函数体时，会销毁 t1 线程的句柄！
}

2. detach()

可以避免线程被销毁，调用detach()意味着线程的生命周期不再由当前 std::thread 对象管理，而是在线程退出以后自动销毁自己。

void myfunc() {
    std::thread t1([&] {
        download("hello.zip");
    });
    t1.detach();
    // t1 所代表的线程被分离了，不再随 t1 对象销毁
}

3. 将即将销毁的线程移动到全局线程池

detach() 的问题是进程退出时候不会等待所有子线程执行完毕。所以另一种解法是把 t1 对象移动到一个全局变量去，从而延长其生命周期到 myfunc() 函数体外。

std::vector<std::thread> pool;

void myfunc() {
    std::thread t1([&] {
        download("hello.zip");
    });
    // 移交控制权到全局的 pool 列表，以延长 t1 的生命周期
    pool.push_back(std::move(t1));
}

int main() {
    myfunc();
    interact();
    for (auto &t: pool) t.join();  // 等待池里的线程全部执行完毕
    return 0;
}

4. 将线程池封装为类

但是需要在 main 里面手动 join 全部线程还是有点麻烦，我们可以自定义一个类 ThreadPool，并用他创建一个全局变量，其解构函数会在 main 退出后自动调用。

class ThreadPool {
    std::vector<std::thread> m_pool;

public:
    void push_back(std::thread thr) {
        m_pool.push_back(std::move(thr));
    }

    ~ThreadPool() {                      // main 函数退出后会自动调用
        for (auto &t: m_pool) t.join();  // 等待池里的线程全部执行完毕
    }
};

ThreadPool tpool;

void myfunc() {
    std::thread t1([&] {
        download("hello.zip");
    });
    // 移交控制权到全局的 pool 列表，以延长 t1 的生命周期
    tpool.push_back(std::move(t1));
}

5. jthread

C++20 引入了 std::jthread 类，和 std::thread 不同在于：他的解构函数里会自动调用 join() 函数，从而保证 pool 解构时会自动等待全部线程执行完毕。

// ~jthread() 解构函数里会自动调用 join()，如果 joinable() 的话
std::vector<std::jthread> pool;

void myfunc() {
    std::jthread t1([&] {
        download("hello.zip");
    });
    // 移交控制权到全局的 pool 列表，以延长 t1 的生命周期
    pool.push_back(std::move(t1));
}

2. async

2.1 异步执行

async()用于启动可异步执行的任务。std::async接受一个带返回值的 lambda，自身返回一个 std::future 对象。get()获取返回值，如果没有执行完毕，则阻塞。

int main() {
    std::future<int> fret = std::async([&] {
        return download("hello.zip"); 
    });
    interact();
    int ret = fret.get(); // 如果此时 download 还没完成，会等待 download 完成，并获取 download 的返回值。
    std::cout << "Download result: " << ret << std::endl;
    return 0;
}

wait()可以等待他执行完，但是不会返回其值。

int main() {
    std::future<int> fret = std::async([&] {
        return download("hello.zip"); 
    });
    interact();
    std::cout << "Waiting for download complete..." << std::endl;
    fret.wait(); // 
    std::cout << "Wait returned!" << std::endl;
    int ret = fret.get();
    std::cout << "Download result: " << ret << std::endl;
    return 0;
}

wait_for()可以指定一个最长等待时间，用 chrono 里的类表示单位。他会返回一个 std::future_status 表示等待是否成功。future_status::timeout或者future_status::ready。还有wait_until()其参数是一个时间点。

int main() {
    std::future<int> fret = std::async([&] {
        return download("hello.zip"); 
    });
    interact();
    while (true) {
        std::cout << "Waiting for download complete..." << std::endl;
        auto stat = fret.wait_for(std::chrono::milliseconds(1000));
        if (stat == std::future_status::ready) {
            std::cout << "Future is ready!!" << std::endl;
            break;
        } else {
            std::cout << "Future not ready!!" << std::endl;
        }
    }
    int ret = fret.get();
    std::cout << "Download result: " << ret << std::endl;
    return 0;
}

2.2 推迟执行

std::async 的第一个参数可以设为 std::launch::deferred，这时不会创建一个线程来执行，他只会把 lambda 函数体内的运算推迟到 future 的 get() 被调用时。

int main() {
    std::future<int> fret = std::async(std::launch::deferred, [&] {
        return download("hello.zip"); 
    });
    interact();
    int ret = fret.get();
    std::cout << "Download result: " << ret << std::endl;
    return 0;
}

2.3 async()的底层实现：future/promise

如果不想让 std::async 帮你自动创建线程，想要手动创建线程，可以直接用 std::promise。

int main() {
    std::promise<int> pret;
    std::thread t1([&] {
        auto ret = download("hello.zip");
        pret.set_value(ret); // 设置返回值
    });
    std::future<int> fret = pret.get_future();

    interact();
    int ret = fret.get(); // 等待并获取线程返回值
    std::cout << "Download result: " << ret << std::endl;

    t1.join();
    return 0;
}

future 为了三五法则，删除了拷贝构造/赋值函数。如果需要浅拷贝，实现共享同一个 future 对象，可以用 std::shared_future。

int main() {
    std::shared_future<void> fret = std::async([&] {
        download("hello.zip"); 
    });
    auto fret2 = fret;
    auto fret3 = fret;
    interact();
    fret3.wait();
    std::cout << "Download completed" << std::endl;
    return 0;
}

3. mutex

<mutex>

3.1 数据竞争

vector不是线程安全的容器，同时访问会出现数据竞争（data-race）。（double free）

int main() {
    std::vector<int> arr;
    std::thread t1([&] {
        for (int i = 0; i < 1000; i++) {
            arr.push_back(1);
        }
    });
    std::thread t2([&] {
        for (int i = 0; i < 1000; i++) {
            arr.push_back(2);
        }
    });
    t1.join();
    t2.join();
    return 0;
}

3.2 mutex、lock()、unlock()

调用 std::mutex 的 lock() 时，会检测 mutex 是否已经上锁:

  • 如果没有锁定，则对 mutex 进行上锁。
  • 如果已经锁定，则陷入等待，直到 mutex 被另一个线程解锁后，才再次上锁。

而调用 unlock() 则会进行解锁操作。

int main() {
    std::vector<int> arr;
    std::mutex mtx;
    std::thread t1([&] {
        for (int i = 0; i < 1000; i++) {
            mtx.lock(); // 
            arr.push_back(1);
            mtx.unlock(); // 
        }
    });
    std::thread t2([&] {
        for (int i = 0; i < 1000; i++) {
            mtx.lock(); // 
            arr.push_back(2);
            mtx.unlock(); //
        }
    });
    t1.join();
    t2.join();
    return 0;
}

3.3 std::lock_guard：符合 RAII 思想的上锁和解锁

根据 RAII 思想，可将锁的持有视为资源，上锁视为锁的获取，解锁视为锁的释放。std::lock_guard 就是这样一个工具类，他的构造函数里会调用 mtx.lock()，解构函数会调用 mtx.unlock()。可以避免不小心忘记解锁。

int main() {
    std::vector<int> arr;
    std::mutex mtx;
    std::thread t1([&] {
        for (int i = 0; i < 1000; i++) {
            std::lock_guard grd(mtx); // 
            arr.push_back(1);
        }
    });
    std::thread t2([&] {
        for (int i = 0; i < 1000; i++) {
            std::lock_guard grd(mtx); // 
            arr.push_back(2);
        }
    });
    t1.join();
    t2.join();
    return 0;
}

3.4 std::unique_lock：也符合 RAII 思想，但自由度更高

1. 直接上锁

lock_guard有时候不太方便，unique_lock额外存储了一个 flag 表示是否已经被释放。他会在解构检测这个 flag，如果没有释放，则调用 unlock()，否则不调用。

int main() {
    std::vector<int> arr;
    std::mutex mtx;
    std::thread t1([&] {
        for (int i = 0; i < 1000; i++) {
            std::unique_lock grd(mtx);
            arr.push_back(1);
        }
    });
    std::thread t2([&] {
        for (int i = 0; i < 1000; i++) {
            std::unique_lock grd(mtx);
            arr.push_back(2);
            grd.unlock(); // 提前解锁
            printf("outside of lock\n");
            // grd.lock();  // 如果需要，还可以重新上锁
        }
    });
    t1.join();
    t2.join();
    return 0;
}

2. 延迟上锁

可以用 std::defer_lock 作为unique_lock的参数，指定了这个参数的话，std::unique_lock 不会在构造函数中调用 mtx.lock()，需要之后再手动调用 grd.lock() 才能上锁。：

int main() {
    std::vector<int> arr;
    std::mutex mtx;
    std::thread t1([&] {
        for (int i = 0; i < 1000; i++) {
            std::unique_lock grd(mtx);
            arr.push_back(1);
        }
    });
    std::thread t2([&] {
        for (int i = 0; i < 1000; i++) {
            std::unique_lock grd(mtx, std::defer_lock); // 
            printf("before the lock\n");
            grd.lock(); // 
            arr.push_back(2);
            grd.unlock(); // 
            printf("outside of lock\n");
        }
    });
    t1.join();
    t2.join();
    return 0;
}

3.5 多个对象，建议每个对象一个锁

不同的对象，各有一个 mutex，独立地上锁，可以避免不必要的锁定，提升高并发时的性能。

int main() {
    std::vector<int> arr1;
    std::mutex mtx1;

    std::vector<int> arr2;
    std::mutex mtx2;

    std::thread t1([&] {
        for (int i = 0; i < 1000; i++) {
            {
                std::lock_guard grd(mtx1);
                arr1.push_back(1);
            }

            {
                std::lock_guard grd(mtx2);
                arr2.push_back(1);
            }
        }
    });
    std::thread t2([&] {
        for (int i = 0; i < 1000; i++) {
            {
                std::lock_guard grd(mtx1);
                arr1.push_back(2);
            }

            {
                std::lock_guard grd(mtx2);
                arr2.push_back(2);
            }
        }
    });
    t1.join();
    t2.join();
    return 0;
}

3.6 try_lock() ：如果上锁失败，不要等待

try_lock()是无阻塞的，他在上锁失败时不会陷入等待，而是直接返回 false；如果上锁成功，则会返回 true。

int main() {
    if (mtx1.try_lock()) // true
        printf("succeed\n");
    else
        printf("failed\n");

    if (mtx1.try_lock()) // false
        printf("succeed\n");
    else
        printf("failed\n");

    mtx1.unlock();
    return 0;
}

3.7 try_lock_for()：只等待一段时间

类似的还有接受时间点的 try_lock_until()。

std::timed_mutex mtx1; // 

int main() {
    if (mtx1.try_lock_for(std::chrono::milliseconds(500))) // 获得锁，返回true
        printf("succeed\n");
    else
        printf("failed\n");

    if (mtx1.try_lock_for(std::chrono::milliseconds(500))) // 等待0.5秒，如果失败就返回false
        printf("succeed\n");
    else
        printf("failed\n");

    mtx1.unlock();
    return 0;
}

3.8 std::unique_lock的参数

1. 用std::try_to_lock作为参数

和无参数相比，他会调用 mtx1.try_lock() 而不是 mtx1.lock()。之后，可以用 grd.owns_lock() 判断是否上锁成功。

int main() {
    std::mutex mtx;
    std::thread t1([&] {
        std::unique_lock grd(mtx, std::try_to_lock); // 
        if (grd.owns_lock()) // 判断是否成功上锁
            printf("t1 success\n");
        else
            printf("t1 failed\n");
        std::this_thread::sleep_for(std::chrono::milliseconds(1000));
    });

    std::thread t2([&] {
        std::unique_lock grd(mtx, std::try_to_lock);
        if (grd.owns_lock())
            printf("t2 success\n");
        else
            printf("t2 failed\n");
        std::this_thread::sleep_for(std::chrono::milliseconds(1000));
    });

    t1.join();
    t2.join();
    return 0;
}

2. 用std::adopt_lock作为参数

如果当前 mutex 已经上锁了，但是之后仍然希望用 RAII 思想在解构时候自动调用 unlock()，可以用 std::adopt_lock 作为 std::unique_lock 或 std::lock_guard 的第二个参数，这时他们会默认 mtx 已经上锁。

int main() {
    std::mutex mtx;
    std::thread t1([&] {
        std::unique_lock grd(mtx);
        printf("t1 owns the lock\n");
        std::this_thread::sleep_for(std::chrono::milliseconds(1000));
    });

    std::thread t2([&] {
        mtx.lock();
        std::unique_lock grd(mtx, std::adopt_lock);
        printf("t2 owns the lock\n");
        std::this_thread::sleep_for(std::chrono::milliseconds(1000));
    });

    t1.join();
    t2.join();
    return 0;
}

3.9 std::unique_lock 和 std::mutex 具有同样的接口

这种只要具有某些指定名字的成员函数，就判断一个类是否满足某些功能的思想，在 Python 称为鸭子类型，而 C++ 称为 concept（概念）。比起虚函数和动态多态的接口抽象，concept 使实现和接口更加解耦合且没有性能损失。

int main() {
    std::mutex mtx;
    std::thread t1([&] {
        std::unique_lock grd(mtx, std::defer_lock);
        std::lock_guard grd2(grd);
        printf("t1 owns the lock\n");
        std::this_thread::sleep_for(std::chrono::milliseconds(1000));
    });

    std::thread t2([&] {
        std::unique_lock grd(mtx, std::defer_lock);
        std::lock_guard grd2(grd);
        printf("t2 owns the lock\n");
        std::this_thread::sleep_for(std::chrono::milliseconds(1000));
    });

    t1.join();
    t2.join();
    return 0;
}

4. 死锁

4.1 同时锁住多个mutex

std::thread t1([&] {
    for (int i = 0; i < 1000; i++) {
        mtx1.lock(); // 
        mtx2.lock();
        mtx2.unlock();
        mtx1.unlock();
    }
});
std::thread t2([&] {
    for (int i = 0; i < 1000; i++) {
        mtx2.lock(); // 
        mtx1.lock();
        mtx1.unlock();
        mtx2.unlock();
    }
});

1. 永远不要同时持有两个锁

如果一个线程永远不要同时持有两个锁，这样可以避免死锁。

std::thread t1([&] {
    for (int i = 0; i < 1000; i++) {
        mtx1.lock(); // 
        mtx1.unlock(); // 
        mtx2.lock();
        mtx2.unlock();
    }
});
std::thread t2([&] {
    for (int i = 0; i < 1000; i++) {
        mtx2.lock();
        mtx2.unlock();
        mtx1.lock();
        mtx1.unlock();
    }
});

2. 保证双方上锁顺序一致

保证双方上锁顺序一致也可以避免死锁。

std::thread t1([&] {
    for (int i = 0; i < 1000; i++) {
        mtx1.lock(); // 
        mtx2.lock();
        mtx2.unlock();
        mtx1.unlock();
    }
});
std::thread t2([&] {
    for (int i = 0; i < 1000; i++) {
        mtx1.lock(); // 
        mtx2.lock();
        mtx2.unlock();
        mtx1.unlock();
    }
});

3. 用std::lock同时上多个锁

它接受任意多个mutex作为参数，并且它保证在无论任意线程中调用的顺序是否相同，都不会产生死锁问题。

std::mutex mtx1;
   std::mutex mtx2;

   std::thread t1([&] {
       for (int i = 0; i < 1000; i++) {
           std::lock(mtx1, mtx2); // 
           mtx1.unlock();
           mtx2.unlock();
       }
   });
   std::thread t2([&] {
       for (int i = 0; i < 1000; i++) {
           std::lock(mtx2, mtx1);
           mtx2.unlock();
           mtx1.unlock();
       }
   });

4.std::lock的RAII版本：std::scoped_lock

和std::lock_guard相对应，只不过它可以同时对多个mutex上锁。

std::thread t1([&] {
    for (int i = 0; i < 1000; i++) {
        std::scoped_lock grd(mtx1, mtx2);
        // do something
    }
});
std::thread t2([&] {
    for (int i = 0; i < 1000; i++) {
        std::scoped_lock grd(mtx2, mtx1);
        // do something
    }
});

4.2 同一个线程重复调用lock()也会造成死锁

void other() {
    mtx1.lock();
    // do something
    mtx1.unlock();
}

void func() {
    mtx1.lock();
    other();
    mtx1.unlock();
}

1. other里不要再上锁

遇到这种情况最好是把 other 里的 lock() 去掉，并在其文档中说明：“other 不是线程安全的，调用本函数之前需要保证某 mutex 已经上锁。”

/// NOTE: please lock mtx1 before calling other()
void other() {
    // do something
}

2. 改用std::recursive_mutex

他会自动判断是不是同一个线程 lock() 了多次同一个锁，如果是则让计数器加1，之后 unlock() 会让计数器减1，减到0时才真正解锁。但是相比普通的 std::mutex 有一定性能损失。

std::recursive_mutex mtx1;

void other() {
    mtx1.lock();
    // do something
    mtx1.unlock();
}

void func() {
    mtx1.lock();
    other();
    mtx1.unlock();
}

5. 数据结构

5.1 多线程环境中使用std::vector

可以封装一个线程安全的vector。（代理模式）

class MTVector {
    std::vector<int> m_arr;
    mutable std::mutex m_mtx;

public:
    void push_back(int val) {
        m_mtx.lock();
        m_arr.push_back(val);
        m_mtx.unlock();
    }

    size_t size() const {
        m_mtx.lock(); // 然而这里因为修改了mutex，导致和const冲突了，所以需要令m_mtx是mutable的
        size_t ret = m_arr.size();
        m_mtx.unlock();
        return ret;
    }
};

int main() {
    MTVector arr;

    std::thread t1([&] () {
        for (int i = 0; i < 1000; i++) {
            arr.push_back(i);
        }
    });

    std::thread t2([&] () {
        for (int i = 0; i < 1000; i++) {
            arr.push_back(1000 + i);
        }
    });

    t1.join();
    t2.join();

    std::cout << arr.size() << std::endl;

    return 0;
}

5.2 读写锁：std::shared_mutex

场景：读可以共享，写必须独占，且写和读不能共存。

class MTVector {
    std::vector<int> m_arr;
    mutable std::shared_mutex m_mtx; // 读写锁

public:
    void push_back(int val) {
        m_mtx.lock(); // 告知为写锁
        m_arr.push_back(val);
        m_mtx.unlock(); // 
    }

    size_t size() const {
        m_mtx.lock_shared(); // 告知为读锁
        size_t ret = m_arr.size();
        m_mtx.unlock_shared(); // 
        return ret;
    }
};

5.3 std::shared_lock：符合RAII思想的lock_shared()

正如 std::unique_lock 针对 lock()，也可以用 std::shared_lock 针对 lock_shared()。这样就可以在函数体退出时自动调用 unlock_shared()，更加安全了。

（std::shared_lock同样支持defer_lock做参数、owns_lock()判断等） 

class MTVector {
    std::vector<int> m_arr;
    mutable std::shared_mutex m_mtx; // 

public:
    void push_back(int val) {
        std::unique_lock grd(m_mtx); // 
        m_arr.push_back(val);
    }

    size_t size() const {
        std::shared_lock grd(m_mtx); // 
        return m_arr.size();
    }
};

5.4 只需一次性上锁，且符合RAII思想：访问者模式

将存储数据的类和访问数据的类分开。

class MTVector {
    std::vector<int> m_arr;
    std::mutex m_mtx;

public:
    class Accessor { // 用于访问数据的类
        MTVector &m_that; // 
        std::unique_lock<std::mutex> m_guard; // 

    public:
        Accessor(MTVector &that) // access()中调用了这个构造函数
            : m_that(that), m_guard(that.m_mtx) // 访问对象初始化时上锁，析构时解锁
        {}

        void push_back(int val) const {
            return m_that.m_arr.push_back(val);
        }

        size_t size() const {
            return m_that.m_arr.size();
        }
    };

    Accessor access() {
        return {*this}; // 利用一个MTVector初始化一个Accessor
    }
};

int main() {
    MTVector arr;

    std::thread t1([&] () {
        auto axr = arr.access(); // 用于访问的类，避免了低效的重复加锁
        for (int i = 0; i < 1000; i++) {
            axr.push_back(i);
        }
    });

    std::thread t2([&] () {
        auto axr = arr.access();
        for (int i = 0; i < 1000; i++) {
            axr.push_back(1000 + i);
        }
    });

    t1.join();
    t2.join();

    std::cout << arr.access().size() << std::endl;

    return 0;
}

5. 条件变量

<condition_variable>

5.1 线程等待被唤醒

std::condition_variable cv; // 
std::mutex mtx; // 需要搭配mutex一起使用

std::thread t1([&] {
    std::unique_lock lck(mtx);
    cv.wait(lck); // 会让当前线程陷入等待，并释放锁

    std::cout << "t1 is awake" << std::endl;
});

std::this_thread::sleep_for(std::chrono::milliseconds(400));

std::cout << "notifying..." << std::endl;
cv.notify_one();  // 会唤醒那个陷入等待的线程

t1.join();

5.2 等待某一条件成真

std::condition_variable cv;
std::mutex mtx;
bool ready = false; // 某个外部条件

std::thread t1([&] {
    std::unique_lock lck(mtx);
    cv.wait(lck, [&] { return ready; }); // 被notify后，只有第二个expr为真（满足条件），才会真正唤醒

    std::cout << "t1 is awake" << std::endl;
});

std::cout << "notifying not ready" << std::endl;
cv.notify_one();  // useless now, since ready = false

ready = true;
std::cout << "notifying ready" << std::endl;
cv.notify_one();  // awakening t1, since ready = true

t1.join();

5.3 多个等待者

之所以条件变量需要一个unique_lock作为参数，是因为要保证多个线程被唤醒时，只有一个能启动。

int main() {
    std::condition_variable cv;
    std::mutex mtx;
	
    //三个线程
    std::thread t1([&] {
        std::unique_lock lck(mtx);
        cv.wait(lck); // 等待
        std::cout << "t1 is awake" << std::endl;
    });

    std::thread t2([&] {
        std::unique_lock lck(mtx);
        cv.wait(lck); // 等待
        std::cout << "t2 is awake" << std::endl;
    });

    std::thread t3([&] {
        std::unique_lock lck(mtx);
        cv.wait(lck); // 等待
        std::cout << "t3 is awake" << std::endl;
    });

    std::this_thread::sleep_for(std::chrono::milliseconds(400));

    std::cout << "notifying one" << std::endl;
    cv.notify_one();  // awakening t1 only

    std::this_thread::sleep_for(std::chrono::milliseconds(400));

    std::cout << "notifying all" << std::endl;
    cv.notify_all();  // awakening t1 and t2

    t1.join();
    t2.join();
    t3.join();

    return 0;
}

5.4 案例：实现生产者-消费者模式

生产者：向缓冲区生产物品，推送后会通知消费者来使用

消费者：等待缓冲区中有物品，没有物品则陷入等待，直到被通知。

int main() {
    std::condition_variable cv;
    std::mutex mtx;

    std::vector<int> foods; // 消费队列

    std::thread t1([&] { // 消费者1
        for (int i = 0; i < 2; i++) {
            std::unique_lock lck(mtx);
            cv.wait(lck, [&] {
                return foods.size() != 0;
            });
            auto food = foods.back();
            foods.pop_back();
            lck.unlock();

            std::cout << "t1 got food:" << food << std::endl;
        }
    });

    std::thread t2([&] { // 消费者2
        for (int i = 0; i < 2; i++) {
            std::unique_lock lck(mtx);
            cv.wait(lck, [&] {
                return foods.size() != 0;
            });
            auto food = foods.back();
            foods.pop_back();
            lck.unlock();

            std::cout << "t2 got food:" << food << std::endl;
        }
    });

    // 生产
    foods.push_back(42);
    foods.push_back(233);
    cv.notify_one();

    foods.push_back(666);
    foods.push_back(4399);
    cv.notify_all();

    t1.join();
    t2.join();

    return 0;
}

5.5 将消费队列封装成类

template <class T>
class MTQueue {
    std::condition_variable m_cv;
    std::mutex m_mtx;
    std::vector<T> m_arr;

public:
    T pop() {
        std::unique_lock lck(m_mtx);
        m_cv.wait(lck, [this] { return !m_arr.empty(); }); // 等待队列不为空
        T ret = std::move(m_arr.back());
        m_arr.pop_back();
        return ret;
    }

    auto pop_hold() {
        std::unique_lock lck(m_mtx);
        m_cv.wait(lck, [this] { return !m_arr.empty(); });
        T ret = std::move(m_arr.back());
        m_arr.pop_back();
        return std::pair(std::move(ret), std::move(lck));
    }

    void push(T val) { // 
        std::unique_lock lck(m_mtx);
        m_arr.push_back(std::move(val));
        m_cv.notify_one(); // 
    }

    void push_many(std::initializer_list<T> vals) {
        std::unique_lock lck(m_mtx);
        std::copy(
                 std::move_iterator(vals.begin()),
                 std::move_iterator(vals.end()),
                 std::back_insert_iterator(m_arr)); // 
        m_cv.notify_all(); // 
    }
};

int main() {
    MTQueue<int> foods;

    std::thread t1([&] {
        for (int i = 0; i < 2; i++) {
            auto food = foods.pop();
            std::cout << "t1 got food:" << food << std::endl;
        }
    });

    std::thread t2([&] {
        for (int i = 0; i < 2; i++) {
            auto food = foods.pop();
            std::cout << "t2 got food:" << food << std::endl;
        }
    });

    foods.push(42);
    foods.push(233);
    foods.push_many({666, 4399});

    t1.join();
    t2.join();

    return 0;
}

5.6 condition_variable tips

std::condition_variable仅仅支持std::unique_lock<std::mutex>作为wait的参数，如果需要使用其他类型的mutex锁，可以用std::conditino_variable_any。

他还有 wait_for() 和 wait_until() 函数，分别接受 chrono 时间段和时间点作为参数。详见：https://en.cppreference.com/w/cpp/thread/condition_variable/wait_for。

6. 原子操作

mutex 太过重量级，他会让线程被挂起，从而需要通过系统调用，进入内核层，调度到其他线程执行，有很大的开销。

6.1 atomic<T>

建议用 atomic：有专门的硬件指令加持，对他的 += 等操作，会被编译器转换成专门的指令。

CPU 识别到该指令时，会锁住内存总线，放弃乱序执行等优化策略（将该指令视为一个同步点，强制同步掉之前所有的内存操作），从而向你保证该操作是原子 (atomic) 的（取其不可分割之意），不会加法加到一半另一个线程插一脚进来。

std::atomic<int> counter = 0; // 

std::thread t1([&] {
    for (int i = 0; i < 10000; i++) {
        counter += 1; // 
    }
});

std::thread t2([&] {
    for (int i = 0; i < 10000; i++) {
        counter += 1; //
    }
});

t1.join();
t2.join();

只支持特定的操作：+=、++，不要把运算符分开。

6.2 fetch_add()相当于+=，但是返回旧值

std::atomic<int> counter;
counter.store(0); // 对应于 =

std::thread t1([&] {
    for (int i = 0; i < 10000; i++) {
        counter.fetch_add(1); // 对应于 += 
    }
});

std::thread t2([&] {
    for (int i = 0; i < 10000; i++) {
        counter.fetch_add(1); // 
    }
});

t1.join();
t2.join();

std::cout << counter.load() << std::endl; // load用于读取其中的int值

int old=atm.fetch_add(val)，这个特点使得他可以用于并行地往一个列表里追加数据。相比于++，它能够追加多个值counter.fetch_add(n)。

std::atomic<int> counter;
counter.store(0);

std::vector<int> data(20000);

std::thread t1([&] {
    for (int i = 0; i < 10000; i++) {
        int index = counter.fetch_add(1); // 除了令counter加1，还会counter返回增加前的值
        data[index] = i; // 追加写入的索引就是 fetch_add 返回的旧值。
    }
});

std::thread t2([&] {
    for (int i = 0; i < 10000; i++) {
        int index = counter.fetch_add(1);
        data[index] = i + 10000;
    }
});

t1.join();
t2.join();

std::cout << data[10000] << std::endl;

6.3 exchange：读取的同时写入

exchange(val) 会把 val 写入原子变量，同时返回其旧的值。

std::atomic<int> counter;

counter.store(0);

int old = counter.exchange(3); // 写入3，返回0
std::cout << "old=" << old << std::endl;  // 0

int now = counter.load();
std::cout << "cnt=" << now << std::endl;  // 3

6.4 compare_exchange_strong：读取，比较是否相等，相等则写入

  • 如果不相等，则把原子变量的值写入 old。
  • 如果相等，则把 val 写入原子变量。

返回一个 bool 值，表示是否相等。

compare_exchange_strong 一般简称 CAS (compare-and-swap)，他是并行编程最常用的原子操作之一。实际上任何 atomic 操作，包括 fetch_add，都可以基于 CAS 来实现。

boolalpha(std::cout);
std::atomic<int> counter;

counter.store(2);

int old = 1;
bool equal = counter.compare_exchange_strong(old, 3); // 2！=1，2写入old，这里old传的是一个引用
std::cout << "equal=" << equal << std::endl;  // false
std::cout << "old=" << old << std::endl;  // 2

int now = counter.load();
std::cout << "cnt=" << now << std::endl;  // 3

old = 2;
equal = counter.compare_exchange_strong(old, 3); // 2==2，3写入counter
std::cout << "equal=" << equal << std::endl;  // true
std::cout << "old=" << old << std::endl;  // 1

now = counter.load();
std::cout << "cnt=" << now << std::endl;  // 3
